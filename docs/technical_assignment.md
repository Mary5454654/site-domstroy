# Техническое задание на создание сайта

## 1. Общие сведения

Наименование системы: информационный сайт строительной компании.

Назначение: публикация информации о строительстве частных домов, услугах, проектах, новостях и способах связи с компанией.

Основание для разработки: задание на практику по созданию сайта на выбранную тематику.

## 2. Назначение и цели создания

Цель разработки - создать простой и удобный сайт, который помогает посетителю ознакомиться с услугами компании и отправить обращение.

Основные задачи:

- представить компанию и направление деятельности;
- разместить проекты домов и описание услуг;
- опубликовать статьи и новости;
- реализовать поиск по сайту;
- добавить форму обратной связи;
- предусмотреть учетные записи пользователей с разными ролями;
- обеспечить стандартный стиль и стиль для слабовидящих.

## 3. Требования к функциональности

Сайт должен содержать:

- не менее 5 и не более 10 основных страниц;
- отдельную страницу карты сайта;
- страницу 404 для несуществующих адресов;
- переключение стандартного режима и режима для слабовидящих;
- меню навигации;
- ленту новостей;
- форму контактов;
- поиск по материалам;
- учетные записи пользователей с ролями;
- сценарий создания пользователя сотрудником.

## 4. Требования к интерфейсу

Интерфейс должен быть понятным для пользователя без дополнительного обучения. Основные разделы должны быть доступны из верхнего меню. Тексты должны быть читаемыми, кнопки и формы - заметными.

Цветовая схема стандартного режима:

- основной цвет: темно-зеленый;
- дополнительный цвет: кирпичный;
- фон: светлый;
- акцент: теплый желтый.

Режим для слабовидящих:

- высокий контраст;
- увеличенный размер шрифта;
- простые цвета;
- заметные границы элементов.

## 5. Требования к программной реализации

Для разработки используется Python и фреймворк Flask. Использование CMS не предусматривается.

Основные компоненты:

- маршруты Flask для страниц сайта;
- шаблоны HTML;
- стили CSS;
- небольшой JavaScript для переключения версии отображения;
- хранение учебных данных в файлах проекта.

## 6. Требования к надежности

При переходе на несуществующую страницу пользователь должен видеть страницу 404. Формы должны проверять обязательные поля.

## 7. Требования к составу документации

В проект входят:

- README с инструкцией по запуску;
- анализ предметной области;
- техническое задание;
- отчет по практике;
- план продвижения;
- анализ экономической эффективности.

## 8. Порядок проверки

Проверяется запуск сайта, доступность основных страниц, работа поиска, переключение режима для слабовидящих, отправка сообщения, вход пользователя и обработка несуществующего адреса.
